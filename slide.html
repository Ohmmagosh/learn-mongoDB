ฐานข้อมูล(DaTabase)

กลุ่มข้อมูลที่ถูกรวบรวมและจัดเก็บไว้อย่างมีโครงสร้างโดยมีรูปแบบการจัดเก็บที่ชัดเจน เป็นระเบียบง่ายต่อการทำไปใช้ตามวัตถุประสงค์ที่ต้องการ

ฐานข้อมูลที่นิยมใช้แบ่งออกเป็น 2 แบบได้แก่

  • ฐานข้อมูลเชิงสัมพันธ์ (Relational Database)
  • ฐานข้อมูลไม่ใช่เชิงสัมพันธ์ (์Non-Relation Database

ฐานข้อมูลเชิงสัมพันธ์(Relational DaTabase)

  • เป็นฐานข้อมูลที่ใช้ภาษา SQL(Structure Query language)มาดำเนินการกับข้อมูล(เพิ่ม,ลบ,แก้ไข,ค้นหา)
  • จัดเก็บข้อมูลแบบตาราง(Table)
  • *กำหนดโครงสร้างการจัดเกบข้อมูลที่ชัดเจน (Structure Data)
  • มีรการเชื่อมโยงความสัมพันธ์กับข้อมูลตารางอื่นด้วย

การปรับขนาด

เมื่อฐานข้อมูลมีการจัดเก็บข้อมูลในปริมาณมาก และอนาคตมีการปรับขนาดฐานข้อมูลให้ใหญ่ขึ้น ฐานข้อมูลเชิงสัมพันธ์จะเป็นการปรับขนาดในแนวตั้ง(Vertical-Scaling)โดยติดตั้งอุปกรณ์เก็บข้อมูลเพื่อเพิ่มความจุ เพิ่มแรมและหน่วยประมวลผลในมีประสิทธิภาพสูงขึ้น ส่งผลให้มีค่าใช้จ่ายที่สูงขึ้นและการอัพเกรดอุปกรณ์ต้องทำการปิดระบบชั่วขณะ

Database Management System

ระบบการจัดการฐานข้อมูล SQL ที่ได้รับความนิยมในปัจจุบัน

  • MySQL
  • Oracle
  • PostgresSQL
  • SQL sever

ฐานข้อมูลไม่ใช่เชิงสัมพันธ์ (Non-Relational Database)

ฐานข้อมูลไม่ใช่เชิงสัมพันธ์หรือเรียกอีกชื่อ คือ NoSQL (Not-Only SQL)ถูกคิดค้นขึ้นเพื่อจัดการข้อจำกัดของการใช้งาน Relational Database ในอดีตเพื่อปรับโครงสร้าง การจัดเก็บข้อมูลให่มีความยืดหยุ่นที่มากขึ้นด้วย

NoSQLถูกนำมาใช้งานการสำหรับประมวลผลข้อมูลจากการใช้งาน Internet อย่างแพร่หลาย ในปัจจุบันไม่วง่าจะมาจากSocial Media ต่างๆที่มีฐานข้อมูลขนาดใหญ่(ฺBig Data)เพื่อแปลข้อมูลที่มีความหลากหลายมากขึ้น เช่น ข้อมูลรูปภาพ วีดีโอ เสียง ตัวอักษร เป็นต้น ซึ่งจะตอบโจทย์ในเรื่องความเร็วความยืดหยุ่นในการจัดเก็บข้อมูล

  • ไม่มีการจัดเก็บข้อมูลในแบบตารางที่มีความสัมพันธ์กัน
  • ไม่มีโครงสร้างการจัดเก็บข้อมูลที่แนนอน(Unstructure Data)
  • ง่ายต่อการปรับขนาดและโครงสร้าง

การปรับขนาด

เมื่อฐานข้อมูลมีการจัดเก็บข้อมูลในปริมาณมาก และอนาคตมีการปรับขนาดฐานข้อมูลให้ใหญ่ขึ้น ฐานข้อมูลไม่ใช่เชิงสัมพันธ์จะเป็นการปรับขนาดในแนวนอน (้Horizontal-Scaling)โดยเพิ่ม Sever เข้าไปแบ่งเบาภาระของข้อมูล สามารถรองรับข้อมูลขนาดใหญ่ขึ้น มีประสิทธฺภาพสูงขึ้น ติดตั้งง่ายและเปลี่ยนแปลงได้ตลอดเวลาโดยไม่ต้องปิดระบบ

รูปแบบการจัดเก็บข้อมูล

NoSQL ไม่มีโครงสร้างการจัดเก็บข้อมูลที่แนนอน เพื่อที่จะสามารถปรับเปลี่ยนโครงสร้างได้ในอนาคตโดยเก็บข้อมูลอยู่ 4 รูปแบบ(ตามลักษณะการใช้งาน)

  • Key - Value Store
  • Document Store
  • Graph Store
  • Wid Column Store
  1. Key - Value Store

    เป็นระบบจัดเก็บข้อมูลในรูปแบบที่เรียกว่า "Associative Array" หรืออาร์เรย์ของชุดข้อมูล ที่มีความสัมพันธ์กันในรูปแบบคู่คีย์(Key:Value)หรือเรียดอีกชื่อว่าDictionary

    โดยคีย์(key)จะทำหน้าที่เป็นตัวเชื่อมโยงไม่ยังค่า(value)ในโครงสร้างของฐานข้อมูลซึ่งจะมีได้แค่คีย์เดียว และการดึงข้อมูลต้องระบุคี่ย์ที่เก็บค่าของข้อมูลนั้นๆไว้

  2. Document Store

    เป็นฐานข้อมูลเชิงเอกสารที่ออกแบบมาเพื่อจัดเก็บข้อมูล ดึงข้อมูล และจัดการข้อมูลที่เรียกว่า"Document-Oriented" คือใช้โครงสร้างภายในของเอกสารเพื่อระบุการจัดเก็บข้อมูล ซึ่งมีลักษณะแบบกึ่งโครงสร้าง(Semi Structure Data) โดยเอกสาร(Document)จะถูกเข้ารหัสข้อมูลและจัดอยู่ในรูปของXML,JSON

  3. Graph Store

    เป็นการจัดเก็บข้อมูลและความสัมพันธ์ของข้อมูลในรูปแบบของกราฟ ข้อมูลด้านในจะเรียกว่าโหนด(Node)ซึ่งจะเก็บ Entity ของข้อมูลและกำหนดความสัมพันธืที่เชื่อมโยงความสัมพันธ์ดังกล่าวผ่าน Edge หารใช้งานกราฟจะถูกนำเสนอผ่านความสัมพันธ์ เช่น Social Network,AI และ Machine Learning เป็นต้น

  4. Wid Column Store

    เป็นการจัดเก็บข้อมูลเป็นเรคอร์ดหรือในรูปแบบตาราง แถว และคอลัมน์ซึ่งจะต่างจาก Relational Database คือจะมีความยืดหยุ่นมากกว่า ชื่อและรูปแบบของคอลัมน์สามารถเปลั่ยนจากแถวนึงไปเป็นอีกแถวนึงในตารางเดียวกันได้ เละขยายคอลัมน์ได้จำนวนมาก(Extensible Record Store:ขยายหารจัดเก็บข้อมูลในแต่ละเรคอร์ด) อาจจะมองเป็นรูปแบบ Key:Value ในรูปแบบ 2 มิติก็ได้

      ตัวอย่างฐานข้อมูลแบบ Wide Column Store
    1. Cassandra
    2. HBase
    3. DynamoDB
    4. Microsoft Azure Cosmos DB

MongoDB คืออะไร

เป็นฐานข้อมูลเชิงเอกสาร (Document Store)สำหรับเก็บข้อมูลขนาดใหญ่ที่มีความยืดหยุ่นสูง ง่ายต่อการปรับขนาดและทำงานข้ามแพลตฟอร์มได้ การจัดเก็บข้อมูลไม่ได้อยู่ในรูปแบบ ตาราง แลจะอยู่ ในรูปแบบเอกสาร json แล้วเซฟเก็บไว้ในเอกสารรูปแบบBinary Bson

รู้จัก JSON และ BSON

json(Javascript Object Notation) ในเริ่มต้นถูกใช้ในการเขียนโปรแกรมด้วยJsvascript ในการแลกเปลี่ยนข้อมูลกับ Sever ปัจจุบัน นิยมนำjson มาใช้เป็นรูปแบบในการแลกเปลี่ยนข้อมูลเว็บหรือแอพพลิเคชั่นมีหลายภาษาที่รองรับการแลกดปลี่ยน ข้อมูลเช่น Java,Python,C# เป็นต้น

    JSON รองรับการทำงานชนิดของข้อมูลพื้นฐานทั้งหมด ได้แก่
  • number:ตัวเลข
  • String:สตริงหรือชุดข้อความโดยใช้เครื่องหมาย Double-quote("")
  • Boolean:ค่าบูลีน(True,False)
  • Null:ค่าว่าง
  • Array:อาร์เรย์หรือชุดข้อมูลซึ่งจะเป็นชนิดข้อมูลใดก็ได้ใช้สัญลักษณ์[]เป็นตัวแสดงและคั่นสมาชิกใน อาร์เรย์แต่ละค่าด้วยคอมม่า(,)เช่น[var1,var2]
  • object:ชุดข้อมูลแบบคู่Key-Value แบบสตริงใช้สัญลักษณ์ปีกกาkey1:value1,key2:value2} และคอมม่าเป็นตัวแบ่งแต่ละคู่ และใช้โคล่อน(:)เป็นตัวแบ่งฝั่งระหว่าง Key และ Value

BSON จะมีขนาดเบาและรับส่งข้อมูลได้เร็ว้หมือนกับJSONโดยMongoDB สามารถเข้าถึงข้อมูลหรือ Object ในBSON เพื่อสร้างIndex ในการทำระบบสอบถามข้อมูล (Query) โดย BSON จะทำงานซ่อนอยู่เบื่องหลัง ส่วนการแสดงข้อมูลกับผู้ใช้จะเป็นJSON
